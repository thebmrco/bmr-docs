# Release 9 (BETA) 2026 February

This release strengthens the overall **platform consistency**, improves reliability in the Discovery workflow, enhances navigation between Library and Room contexts, and introduces the first step toward spatially aware acoustic documentation inside the Digital Twin.

The update also aligns the Web interface with the new design language introduced in the App.

---

## App Improvements

<summary className="bmr-summary">

The App now adopts the updated BMR design language, aligned with the Web platform.

**What’s improved**
- Updated icon set consistent with the Web interface  
- Refined icon sizing and visual balance  
- Improved visual hierarchy  
- Cleaner navigation elements  
- More consistent spacing and typography  

This creates a unified experience across App and Web, reducing cognitive friction when switching between devices and strengthening platform consistency.

<details className="bmr-details">
<summary className="bmr-summary">
  <span style={{display:"inline-flex", alignItems:"center", gap:"10px"}}>
    <CloudUpload size={18} /> Extended Resilience During Data Upload
  </span>
</summary>

Uploading scan, marker, and acoustic data is now significantly more robust.

**New behavior**
- Data uploads automatically in the background  
- A status banner clearly indicates upload progress  
- If upload fails completely, data is safely stored locally  
- Users can manually retry uploads at a later time  

This ensures:
- No data loss due to unstable network conditions  
- Reduced workflow interruptions  
- Improved reliability in field environments  

</details>

<details className="bmr-details">
<summary className="bmr-summary">
  <span style={{display:"inline-flex", alignItems:"center", gap:"10px"}}>
    <ArrowRightSquare size={18} /> Direct Access to Room from Library
  </span>

Users can now navigate directly to the associated room from Library items.

Supported items:
- Acoustic measurements  
- Scans  
- Designs  
- Markers  

This eliminates unnecessary back-navigation and improves workflow efficiency when reviewing historical data or continuing work within a specific room context.

</details>

---

<details className="bmr-details">
<summary className="bmr-summary">
  <span style={{display:"inline-flex", alignItems:"center", gap:"10px"}}>
    <MapPin size={18} /> Acoustic Test Records Position Coordinates (Digital Twin Validation)
  </span>
</summary>

Acoustic measurements now capture spatial position data when performed inside a room containing a Digital Twin.

**New functionality**
- The phone’s position is recorded when the acoustic test begins  
- The system verifies the presence of a Digital Twin before enabling spatial recording  
- Position data will be visualized inside the Digital Twin  

This marks the first step toward a fully spatial acoustic documentation flow, where speaker + measurement locations can be visualized and compared over time.

</details>
