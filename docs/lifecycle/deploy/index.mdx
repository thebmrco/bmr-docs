# <Icon name="circle-check" size="1em" /> Deploy Phase

The **Deploy Phase** is when an approved design transitions from plan to reality. Once a design is physically installed, it is marked as **Active** in BMR Studio — becoming the authoritative record of the room as it currently exists.

---

## <Icon name="circle-check" size="1em" /> The Active Design

Within BMR Studio, each room can have multiple design versions. The design that has been approved and physically installed is marked as the **Active Design**.

The Active Design is:

- The **authoritative record** of the room as it stands today
- The **single source of truth** for equipment, layout, cable routing, and infrastructure
- Equivalent to a **digital twin** of the physical room
- The reference used by facilities, IT, and support teams going forward

All other design versions represent proposed or historical states. Only one design can be the Active Design at a time.

:::info
The Active Design is tracked automatically as designs move through the approval and deployment workflow in BMR Studio. You do not need to maintain a separate document — the platform keeps the record up to date.
:::

---

## <Icon name="circle-check" size="1em" /> Deployment Outputs

When a design is approved and ready for deployment, it can include:

- **Detailed equipment list** — all components placed in BMR Studio, structured and ready for procurement or handover
- **Structured cabling guide** — cable types, lengths, and routing logic derived from the design
- **Coverage visuals** — camera, screen, and audio coverage for reference during installation
- **Full version history** — all prior design versions remain accessible for audit and reference

This ensures a clear, unambiguous handover from design to installation, reducing deployment risk.

---

## <Icon name="palette" size="1em" /> Marking a Design as Active

When installation is complete:

1. Open the room in **BMR Studio**
2. Select the design version that matches the installed state
3. Move it through the **approval workflow**
4. Mark it as **Active**

The design is now the Active Design — the room's live record. Future changes to the room start a new design version, which becomes the new Active Design once installed.

---

## <Icon name="circle-check" size="1em" /> What the Active Design enables

- **Maintenance** — technicians and facilities teams see exactly what is installed, where, and how it is connected
- **Upgrades** — new design iterations are built on top of the current Active Design
- **Compliance & governance** — portfolio managers can view the active state of every room in the organization

---

## <Icon name="alert-triangle" size="1em" /> Prerequisites

- ✅ Completed **design** in BMR Studio with equipment, layout, and cabling
- ✅ Design has been **approved** through the review workflow
- ✅ Physical **installation** is complete and matches the design
